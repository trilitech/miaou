Course 2

Adapt to terminal width.

Terminal applications need responsive layouts too. This lesson builds on Course 1: your page already receives size, so it can choose a layout for the current terminal width.

You will learn: breakpoint ordering, cheap layout selection, and how to keep width decisions inside render-time data flow.

How it works

Responsive.pick walks breakpoints in order and returns the first layout whose max_width is greater than or equal to the current width. In the example above, width 80 selects the medium two-by-two grid rendered by render_dashboard.

Recommended pattern

Define layout variants as data first, then render the selected variant. Keep the breakpoint decision cheap because it normally runs from view.
